# MVP — StartDev Valley (2.5D)
### Gra zachęcająca do nauki programowania

---

## Cel MVP
Dostarczyć działające demo gry 2.5D, w którym gracz:
- przechodzi tutorial uczący podstaw: zmienne, instrukcje warunkowe (`if`), pętle,
- rozgrywa jeden prosty poziom z przeciwnikami i zagadkami/questami logicznymi,
- korzysta z systemu walki i eksploracji sterowanego kodem (C++ lub Python).

---

## Zakres funkcjonalny (In scope)
- **Silnik gry 2.5D:** ruch postaci, kamera, kolizje, interakcje (otwieranie drzwi, dźwignie, skrzynie).
- **Edytor/konsola kodu w grze:** okno do pisania krótkich snippetów, uruchamianie, podpowiedzi składni, komunikaty błędów.
- **Dwa języki wykonawcze:** wybór C++ lub Python przy starcie (prosty sandbox z bezpiecznymi API gry).
- **Tutorial:**
  - zmienne (liczbowe/logic), wejście/wyjście,
  - `if / else`, operatory porównania,
  - pętle (`for`/`while`) na przykładach sterujących NPC/urządzeniami.
- **Poziom 1:**
  - przeciwnicy: proste AI, podatne na skrypty gracza
  - questy: proste zadania fabularne z zagadkami, które wprowadzają proste mechaniki i zasady programowanie w pythonie/C++
- **UI/UX:** HUD (HP/energia, wskaźnik celu), log zdarzeń, podpowiedzi w tutorialu.
- **Progres:** zapis/odczyt postępów (checkpointy).
- **Audio/Wizualia:** podstawowe efekty (kroki, trafienia, trigger zagadek), placeholdery assetów.

---

## Poza zakresem (Out of scope)

- Ekonomia, crafting, rozbudowana fabuła, tryb wieloosobowy,
- rozbudowane drzewka umiejętności, pełna lokalizacja, zaawansowane AI.

---

## Kluczowe mechaniki „programowalne”

- **Walki/eksploracja przez kod:**
 wywoływanie prostych API: atak, blok, użycie przedmiotu
 - **Interfejs kodowania:** mini-IDE z walidacją, limitem czasu/CPU, przykładowymi snippetami, przyciskiem „Uruchom”.
- **Bezpieczeństwo:** sandbox (brak I/O poza API gry, limit pamięci/czasu, whitelisting bibliotek).

---

## Treści edukacyjne w tutorialu

- **Zmienne:** typy, przypisanie, zakres podstawowy.  
- **Warunki:** `if/else`, operatory (`==`, `!=`, `<`, `>`, `&&/and`, `||/or`).  
- **Pętle:** `for`, `while`, `break`, licznik kroków.  
- **Funkcje:** definicja i wywołanie jednej prostej funkcji użytkownika.

---

## Akceptacja (Acceptance Criteria)

- Gracz może ukończyć tutorial bez pomocy z zewnątrz (wbudowane wskazówki).  
- Oba języki działają w tych samych zadaniach; różni się tylko składnia.  
- System zgłasza czytelne błędy (kompilacja/wykonanie) i nie crashuje gry.  
- Zapis postępu działa (co najmniej 1 checkpoint).  
- FPS ≥ 30 na docelowej platformie testowej; czas uruchomienia kodu ≤ 100 ms dla typowych snippetów.  
- Brak dostępu kodu do dysku/sieci; sandbox egzekwuje limity.

---

## Metryki sukcesu

- Ukończenie tutorialu ≥ 80% graczy testowych. 
- Błędy krytyczne (crashe, deadlocki sandboxa): 0 w sesji testowej.  
- Czas sesji MVP: 20–30 min.

---

## Definition of Done (DoD)

- Tutorial działa zgodnie z dokumentacją; testy API pokrywają happy path i 1–2 edge cases.  
- Brak błędów krytycznych, logi ostrzeżeń przejrzane.  
- UI responsywne, komunikaty zrozumiałe.  
- Zaktualizowana dokumentacja (`README` „Jak grać” + notatki dla QA).

---

## Materiały / placeholdery

- Proste modele 2.5D/tilesety, dźwięki (kroki/uderzenia), ikony UI (open-source/placeholder).

---

## Wynik MVP

Jedno-plikowe demo z wyborem języka (C++/Python), tutorialem i poziomem 1, w którym gracz rozwiązuje zagadki i walczy za pomocą kodu, kończąc sesję ekranem podsumowania.
